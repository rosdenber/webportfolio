Reflection

What technologies did you use and why?

For this portfolio, I used HTML, CSS (with Tailwind CSS), and JavaScript because these are the technologies I am most familiar with. They allowed me to build a responsive, interactive, and visually appealing interface. HTML provided the structure, Tailwind CSS made styling faster and more consistent, and JavaScript added the interactivity that brings the portfolio to life.

Which parts of the portfolio were easiest for you?

The easiest part for me was building the HTML structure and creating the content. I’m comfortable with markup and organizing information, so writing and arranging each section felt natural and allowed me to showcase my projects and skills clearly.

Which parts were most challenging?

The most challenging part of my portfolio was working with Tailwind CSS. Since I’m more used to traditional frameworks like Bootstrap, I had to adjust how I think about styling and change my usual workflow. It took some time to get used to the utility-first approach, but it helped me learn new techniques and improve my skills.

What technologies are you most interested in learning?

I’m particularly interested in learning mobile application development. I’m fascinated by how apps are built, how the frontend and backend communicate, and how data is processed and stored securely within the system. Understanding these details would complement my web development skills and give me a broader perspective on building apps that work seamlessly across platforms.

What kind of applications do you want to build in the future?

In the future, I want to build game applications that are not only for entertainment but also meaningful. I want to create games that go beyond entertainment by teaching valuable skills, promoting learning, or providing practical solutions for everyday life.
